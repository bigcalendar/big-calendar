# useMonthView

A hook that composes all the logic needed to render a custom month grid view.

## What it returns

- **`grid`** — the rendered month model: weeks, day cells, event segments, and
  overflow data. `null` when the active view is not the month (use as a gate).
- **`components`** — resolved slot components (`Weekday`, `DateCell`,
  `EventSlot`, `ShowMore`). Defaults or overrides from `components.month`.
- **`messages`** — localized strings (e.g. the "show more" label).
- **`announcement`** — a string for the keyboard-DnD live region (empty when
  idle).
- **`root`** / **`monthGrid`** — element-spread props for the outer `<div>`
  and the CSS grid `<div>`. Spread these directly onto their elements — they
  carry className, ref, and all keyboard/pointer handlers.
- **`getDaySlotProps(cell, weekIndex, dayIndex)`** — returns element-spread
  props for the invisible slot hit-target inside each day cell (used for slot
  selection and roving focus).
- **`getWeekSelectionBand(weekIndex)`** — returns the selection highlight for
  a week row, or `null` when no selection covers it.
- **`getWeekPreviewBand(week)`** — returns the drag-preview highlight for a
  week row, or `null` when no preview covers it.
- **`getSegmentProps(segment)`** — returns all `EventButton` props for a
  placed event segment.
- **`getShowMoreCellProps(cell, dayIndex, moreRow)`** — returns wrapper and
  `ShowMore` props for an overflowed day, or `null` when there is no overflow.

## How to use it

Call the hook inside a component that lives inside a `CalendarProvider`:

```tsx
import { EventButton, useMonthView } from '@big-calendar/react'

function MyMonthView() {
  const { grid, components, root, monthGrid, getDaySlotProps, getSegmentProps } =
    useMonthView()

  if (grid === null) return null

  const { Weekday, DateCell, EventSlot } = components

  return (
    <div {...root}>
      <div className="bc-month-header">
        {grid.weekdays.map((wd) => (
          <Weekday key={wd.day} day={wd.day} long={wd.long} short={wd.short} />
        ))}
      </div>
      <div {...monthGrid}>
        {grid.weeks.map((week, wi) => (
          <div key={week.key} className="bc-month-week">
            <div className="bc-month-slots">
              {week.days.map((cell, di) => (
                <div key={cell.day} {...getDaySlotProps(cell, wi, di)} />
              ))}
            </div>
            <div className="bc-week-backgrounds">
              {week.days.map((cell) => (
                <DateCell key={cell.day} {...cell} onDrillDown={() => {}} />
              ))}
            </div>
            <div className="bc-week-events">
              {week.segments.map((seg) => (
                <EventButton key={seg.key} {...getSegmentProps(seg)}>
                  <EventSlot event={seg.event} title={seg.title} />
                </EventButton>
              ))}
            </div>
          </div>
        ))}
      </div>
    </div>
  )
}
```

## When to use this hook vs. `MonthView`

Use `MonthView` when the default rendering is good enough. Use `useMonthView`
when you need to change the HTML structure — for example, to render the grid
as a `<table>`, to add custom row wrappers, or to change how segments are
positioned. For changing individual cells or labels, use the `components.month`
slot overrides instead (no hook required).

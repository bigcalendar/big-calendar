# Popover

`Popover` is an **anchored top-layer surface**. It is the building block behind the
month and all-day "+N more" indicators, and is exported for your own popovers.

## Renders in the top layer

The panel uses the native **Popover API** (`popover="auto"` + the trigger's
`popovertarget`). It is promoted to the browser **top layer**, so it draws above
every other element and is not clipped by an ancestor's `overflow` or trapped by a
stacking context. The browser provides **light-dismiss** (click outside), **Esc to
close**, and correct toggle behavior for free — React only tracks open state (via
the `toggle` event) and positions the panel.

## Positioning & fallback

Placement is computed by **`@floating-ui/core`**, not CSS Anchor Positioning. Anchor
Positioning is not yet cross-engine (Chromium-only as of this writing), so floating-ui
is the **default** positioning engine for all tethered top-layer surfaces (§7.5). It is
**lazy-loaded** — the module is dynamically imported the first time a popover opens, so
it stays out of the initial bundle. Native `anchor()` may be adopted later as a
feature-detected progressive enhancement once it ships across engines.

## Accessibility

The trigger is wired with `aria-haspopup="dialog"`, `aria-expanded`, and
`aria-controls`. Because it is built on `popovertarget`, the trigger should be a
`<button>`.

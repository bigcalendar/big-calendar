# useTimeGridView

A composed hook that drives a custom time-grid view (Day, Week, Work-Week).

## What it returns

- **`grid`** — the resolved time grid model, or `null` when the active view is
  not a time-grid view (use as a gate). When non-null, it contains the column
  headings, all-day row data, time columns, gutter labels, and the slot count.
- **`announcement`** — a string for the keyboard-DnD live region (empty when
  idle). Render it inside a `role="status"` live region that is always mounted.
- **`root`** — element-spread props for the outermost `<div>`: the event-roving
  `ref` and keyboard-DnD handlers.
- **`getRootStyle(leafCount)`** — returns a CSS style object for the root
  `<div>`. Pass the number of "leaf" columns (one per day in plain mode).
- **`header`** — everything needed to render the header section (day headings,
  all-day row). See `useTimeGridHeader`.
- **`body`** — everything needed to render the scrollable body (time columns,
  event boxes, gutter). See `useTimeGridBody`.

## Composed vs. granular hooks

`useTimeGridView` is a convenience wrapper. Internally it calls:

1. **`useTimeGrid()`** — builds the grid model from the store.
2. **`useTimeGridHeader(grid)`** — header state, allday-row interaction,
   slot selection, roving focus, and allday-segment geometry.
3. **`useTimeGridBody(grid)`** — body state, time-slot interaction, event
   positioning, background events, the now indicator, and slot selection for
   the body.

If you only need to replace the header or the body, call those hooks directly
instead. `useTimeGridView` is the right choice when you want to replace the
entire time-grid layout.

## Layout variants

The time-grid view has three layout paths:

- **Plain** — `grid.resources === null && grid.dayGroups === null`. Standard
  Day / Week / Work-Week with one column per day.
- **Resource-major** — `grid.resources !== null`. Each resource has its own
  group of day columns; the header has optional two-tier resource + day labels.
- **Day-major** — `grid.dayGroups !== null`. Columns are ordered day-first,
  so all resources for Monday appear before any resources for Tuesday.

The story on this page covers the plain layout. For resource layouts, see the
`TimeGridView` stories.

## How to use it

```tsx
import { EventButton, selectionStyle, useTimeGridView } from '@big-calendar/react'

function MyTimeGridView() {
  const { grid, announcement, root, getRootStyle, header, body } =
    useTimeGridView()

  // Return nothing when a different view is active.
  if (grid === null) return null

  // This example covers the plain (non-resource) layout only.
  const { components: { DayHeading }, getHeadingProps, allDayRow } = header
  const { components: { TimeLabel, EventSlot }, body: bodyRoot, getColumnProps, getSlotProps, getEventProps } = body

  return (
    <div className="bc-time-grid" style={getRootStyle(grid.headings.length)} {...root}>
      <div className="bc-sr-only" role="status" aria-live="polite">
        {announcement}
      </div>
      <div className="bc-time-header">
        <div className="bc-time-header-gutter" aria-hidden="true" />
        {grid.headings.map((h) => (
          <DayHeading key={h.day} {...getHeadingProps(h)} />
        ))}
      </div>
      <div className="bc-allday-row" {...allDayRow}>
        {/* ... all-day segments */}
      </div>
      <div {...bodyRoot}>
        <div className="bc-time-gutter">
          {grid.gutter.map((l) => (
            <TimeLabel key={l.key} time={l.time} label={l.label} />
          ))}
        </div>
        {grid.columns.map((column, colIndex) => (
          <div key={column.key} {...getColumnProps(column)}>
            <div className="bc-time-slots">
              {Array.from({ length: grid.slotCount }, (_, si) => (
                <div key={si} {...getSlotProps(column, colIndex, si)} />
              ))}
            </div>
            {column.events.map((event) => (
              <EventButton key={event.key} {...getEventProps(event)}>
                <EventSlot event={event.event} title={event.title} time={event.time} />
              </EventButton>
            ))}
          </div>
        ))}
      </div>
    </div>
  )
}
```

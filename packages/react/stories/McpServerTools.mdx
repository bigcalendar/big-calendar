# MCP Server — Tools Reference

The `@big-calendar/mcp` server exposes six tools, ten resources, and two prompts.
Tools take arguments and return generated code or status text.
Resources are read-only documents your AI assistant can pull in as context.
Prompts are conversation starters that chain tools together.

---

## Tools

### init

Checks for an existing `bc.md` memory file or writes a new one.

**With no arguments**, returns a status report of the current `bc.md` (or a warning if it does not exist).

**With any content argument**, writes or overwrites `bc.md` with the provided values.

### scaffold-calendar

Generates a ready-to-use `Calendar` component using your `bc.md` configuration.

The generated component includes:
- `createTemporalLocalizer` setup (no extra dependencies required)
- `CalendarProvider` with your `accessors` wired to the correct field names
- All enabled `views` in the provider's view list
- `useCalendarDnd` + `CalendarWithDnd` wrapper if `dnd` is listed in features

### generate-sample-events

Generates a TypeScript interface and a typed `const` array of sample events using your `bc.md` accessor mapping.

Field names in the output match the accessor values from `bc.md`, not the Big Calendar key names.
For example, if `title: name` is set in your accessor map, the generated interface has a `name: string` field, not `title`.

## Prompts

### bootstrap-calendar

Runs the full onboarding flow in a single prompt: checks for `bc.md`, collects your event field names and feature choices, writes `bc.md`, then runs `scaffold-calendar` to generate a starter component.

**No arguments required.** Invoke via your client's prompt picker.

### add-feature

Opens a guided flow to add one feature (`dnd`, `selection`, or `resources`) to an existing integration.
Optionally accepts a `feature` argument to skip the selection step.

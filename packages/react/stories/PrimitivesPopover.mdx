# Popover

A top-layer floating panel built on the browser's native [Popover API](https://developer.mozilla.org/en-US/docs/Web/API/Popover_API). The element renders in the browser's top layer, above all other content, with no stacking-context or z-index issues.

## Where it is used

Big Calendar uses `<Popover>` for the month grid's **show-more** overlay (the panel that lists overflowed events when you click "+N more") and the time-grid's **all-day show-more** overlay. It is also available for use in your own custom slot components.

## API

```tsx
import { Popover } from '@big-calendar/react'

<Popover
  placement="bottom-start"   // floating-ui placement string
  sameWidth                  // match the trigger's rendered width
  className="my-popover"
  trigger={<button>Open</button>}
>
  <p>Popover content</p>
</Popover>
```

## Positioning

`<Popover>` uses `@floating-ui/core` (lazily loaded) with `offset`, `flip`, `shift`, and `size` middleware to keep the panel within the viewport. The available height is written as a CSS custom property (`--bc-float-avail-h`) on the panel element, so your CSS can constrain height with `min(Npx, var(--bc-float-avail-h))` without being overridden by inline styles.

## Styling

The base class is `bc-popover` (from `@big-calendar/styles`). The show-more variant adds `bc-show-more-popover`, which caps height at `min(300px, var(--bc-float-avail-h, 100vh))` with vertical scroll.

Override appearance via CSS custom properties:

```css
.bc-popover {
  --bc-radius: 8px;
  --bc-shadow-popover: 0 4px 24px rgb(0 0 0 / 0.12);
}
```

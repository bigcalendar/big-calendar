# useTimeGridView

A composed hook that drives a custom time-grid view (Day, Week, Work-Week). Call it inside a component that lives inside a `CalendarProvider`. See <LinkTo title="Advanced/Headless API" story="docs">Advanced → Headless API</LinkTo> for the general pattern.

## Layout variants

The time grid has three layout paths, identified by checking the `grid` object:

<table>
  <thead><tr><th>Condition</th><th>Layout</th></tr></thead>
  <tbody>
    <tr><td><code>grid.resources === null &amp;&amp; grid.dayGroups === null</code></td><td><strong>Plain</strong> — one column per day.</td></tr>
    <tr><td><code>grid.resources !== null</code></td><td><strong>Resource-major</strong> — resources as the outer grouping.</td></tr>
    <tr><td><code>grid.dayGroups !== null</code></td><td><strong>Day-major</strong> — days as the outer grouping (<code>resourceLayout="day"</code>).</td></tr>
  </tbody>
</table>

## Composed vs. granular hooks

`useTimeGridView` wraps three lower-level hooks:

1. **`useTimeGrid()`** — builds the grid model from the store.
2. **`useTimeGridHeader(grid)`** — header state, all-day-row interaction, roving focus, segment geometry.
3. **`useTimeGridBody(grid)`** — body state, time-slot interaction, event positioning, background events, now indicator.

Call those directly if you only need to replace the header or body, not the whole layout.

## Usage example (plain layout)

```tsx
import { EventButton, useTimeGridView } from '@big-calendar/react'

function MyTimeGridView() {
  const { grid, announcement, root, getRootStyle, header, body } =
    useTimeGridView()

  if (grid === null) return null

  const {
    components: { DayHeading, AllDayEvent },
    timeHead, timeHeader, timeHeaderGutter,
    allDayRow, allDayLabel, allDaySlots, allDaySegments,
    getHeadingProps, getAllDaySlotProps, getAllDaySegmentProps,
  } = header

  const {
    components: { TimeLabel, EventSlot },
    body: bodyRoot, gutter, timeSlotsContainer,
    getColumnProps, getSlotProps, getEventProps,
    bodyNowIndicatorProps,
  } = body

  return (
    <div style={getRootStyle(grid.headings.length)} {...root}>
      <div role="status" aria-live="polite" className="bc-sr-only">
        {announcement}
      </div>
      <div {...timeHead}>
        <div {...timeHeader}>
          <div {...timeHeaderGutter} />
          {grid.headings.map((h) => (
            <DayHeading key={h.day} {...getHeadingProps(h)} />
          ))}
        </div>
        <div {...allDayRow}>
          <div {...allDayLabel}>All day</div>
          <div {...allDaySlots}>
            {grid.columns.map((col, ci) => (
              <div key={col.key} {...getAllDaySlotProps(col, ci)} />
            ))}
          </div>
          <div {...allDaySegments}>
            {grid.allDay.segments.map((seg) => (
              <EventButton key={seg.key} {...getAllDaySegmentProps(seg)}>
                <AllDayEvent event={seg.event} title={seg.title} />
              </EventButton>
            ))}
          </div>
        </div>
      </div>
      <div {...bodyRoot}>
        <div {...gutter}>
          {grid.gutter.map((l) => (
            <TimeLabel key={l.key} time={l.time} label={l.label} />
          ))}
        </div>
        {grid.columns.map((column, ci) => (
          <div key={column.key} {...getColumnProps(column)}>
            <div {...timeSlotsContainer}>
              {Array.from({ length: grid.slotCount }, (_, si) => (
                <div key={si} {...getSlotProps(column, ci, si)} />
              ))}
            </div>
            {column.events.map((event) => (
              <EventButton key={event.key} {...getEventProps(event)}>
                <EventSlot event={event.event} title={event.title} time={event.time} />
              </EventButton>
            ))}
          </div>
        ))}
        {bodyNowIndicatorProps && <div {...bodyNowIndicatorProps} />}
      </div>
    </div>
  )
}
```

## When to use this vs. `TimeGridView`

Use `TimeGridView` when the default rendering is sufficient. Use `useTimeGridView` when you need to change the HTML structure — render the all-day row differently, replace the column layout, or add custom wrappers.

For changing individual cells or labels without restructuring, use `components.time` slot overrides — no hook required.

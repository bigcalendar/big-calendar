# useAgendaView

A hook that composes all the data and element-spread props needed to render a custom agenda view. Call it inside a component that lives inside a `CalendarProvider`. See <LinkTo title="Advanced/Headless API" story="docs">Advanced → Headless API</LinkTo> for the general pattern.

## Return value

<table>
  <thead><tr><th>Property</th><th>Type</th><th>Description</th></tr></thead>
  <tbody>
    <tr><td><code>rows</code></td><td><code>AgendaRow[] | null</code></td><td>The list of day groups to render, each with a date and its events. <code>null</code> when the active view is not the agenda — use as a gate.</td></tr>
    <tr><td><code>components</code></td><td><code>object</code></td><td>Resolved slot components: <code>DateSlot</code>, <code>EventSlot</code>, <code>EmptySlot</code>.</td></tr>
    <tr><td><code>messages</code></td><td><code>Messages</code></td><td>Localized strings for column headings and the empty state.</td></tr>
    <tr><td><code>root</code></td><td><code>object</code></td><td>Element-spread props for the outer <code>&lt;div&gt;</code>.</td></tr>
    <tr><td><code>header</code></td><td><code>object</code></td><td>Element-spread props for the sticky heading row.</td></tr>
    <tr><td><code>body</code></td><td><code>object</code></td><td>Element-spread props for the scrollable list.</td></tr>
    <tr><td><code>getRowProps(row)</code></td><td><code>function</code></td><td>Returns <code>className</code> and a CSS <code>style</code> for each day-group <code>&lt;div&gt;</code>.</td></tr>
  </tbody>
</table>

## Usage example

```tsx
import { useAgendaView } from '@big-calendar/react'

function MyAgendaView() {
  const { rows, components, messages, root, header, body, getRowProps } =
    useAgendaView()

  if (rows === null) return null

  const { DateSlot, EventSlot, EmptySlot } = components

  return (
    <div {...root}>
      <div {...header}>
        <span>{messages.date}</span>
        <span>{messages.time}</span>
        <span>{messages.event}</span>
      </div>
      {rows.length === 0 ? (
        <EmptySlot message={messages.noEventsInRange} />
      ) : (
        <div {...body}>
          {rows.map((row) => (
            <div key={row.day} {...getRowProps(row)}>
              <DateSlot day={row.day} label={row.label} />
              {row.events.map((item) => (
                <EventSlot
                  key={item.key}
                  event={item.event}
                  title={item.title}
                  time={item.time}
                  allDay={item.allDay}
                />
              ))}
            </div>
          ))}
        </div>
      )}
    </div>
  )
}
```

Place your component inside `CalendarProvider`:

```tsx
<CalendarProvider defaultView="agenda" events={events}>
  <Toolbar />
  <div className="bc-calendar">
    <MyAgendaView />
  </div>
</CalendarProvider>
```

## When to use this vs. `AgendaView`

Use `AgendaView` when the default rendering is sufficient. Use `useAgendaView` when you need to change the HTML structure — render events in a `<table>`, add custom wrapper elements, or change how the date and event columns are laid out.

For changing individual cells or labels without restructuring, use `components.agenda` slot overrides — no hook required.

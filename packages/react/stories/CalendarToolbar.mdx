# Toolbar

The default toolbar provides **navigation** (Prev / Today / Next buttons) and a **view switcher** (one button per enabled view). It lives outside the `.bc-calendar` element so its height does not shrink the calendar's scrollable body.

## Hiding the toolbar

Pass `toolbar={false}` to `<Calendar>` to remove it:

```tsx
<Calendar toolbar={false} />
```

Useful when your application provides its own navigation controls or when embedding a calendar widget without chrome.

## Customizing labels and messages

The toolbar reads from the `messages` prop on `CalendarProvider`. Override any string:

```tsx
<CalendarProvider
  messages={{
    today: 'Jump to today',
    previous: '‹',
    next: '›',
    month: 'Month',
    week: 'Week',
    work_week: 'Work',
    day: 'Day',
    agenda: 'Agenda',
  }}
>
```

## Replacing the toolbar entirely

Provide your own component via `components.toolbar`. It receives the same props as the default toolbar:

```tsx
import type { ToolbarProps } from '@big-calendar/react'

function MyToolbar({ label, view, views, messages, onNavigate, onView }: ToolbarProps) {
  return (
    <header>
      <button onClick={() => onNavigate('PREV')}>Previous</button>
      <span>{label}</span>
      <button onClick={() => onNavigate('NEXT')}>Next</button>
      <button onClick={() => onNavigate('TODAY')}>Today</button>
      <nav>
        {views.map((v) => (
          <button key={v} onClick={() => onView(v)} aria-pressed={v === view}>
            {messages[v] ?? v}
          </button>
        ))}
      </nav>
    </header>
  )
}

<CalendarProvider components={{ toolbar: MyToolbar }}>
  <Calendar />
</CalendarProvider>
```

## Using the Toolbar component standalone

`<Toolbar>` is exported and can be placed anywhere inside a `CalendarProvider` — it does not have to be inside `<Calendar>`. This is how the `Calendar` component itself uses it:

```tsx
<CalendarProvider ...>
  <Toolbar />             {/* navigation lives here, outside .bc-calendar */}
  <div className="bc-calendar">
    <MonthView />
    <TimeGridView />
    <AgendaView />
  </div>
</CalendarProvider>
```

If you are building a custom layout with the headless hooks, this gives you full control over where navigation sits relative to the calendar body.

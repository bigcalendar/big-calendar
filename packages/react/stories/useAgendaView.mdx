# useAgendaView

A hook that composes all the data and element-spread props needed to render a
custom agenda view.

## What it returns

- **`rows`** — the list of day groups to render, each with a date and its
  events. `null` when the active view is not the agenda (use as a gate).
- **`components`** — the resolved slot components (`DateSlot`, `EventSlot`,
  `EmptySlot`). These are either the defaults or whatever was passed via
  `components.agenda` on `CalendarProvider`.
- **`messages`** — localized strings for the column headings and empty state.
- **`root` / `header` / `body`** — element-spread props (just a `className`)
  to apply to the outer div, the sticky heading row, and the scrollable list.
- **`getRowProps(row)`** — returns `className` and a CSS `style` for each
  day-group div.

## How to use it

Call the hook inside a component that lives inside a `CalendarProvider`. Spread
the returned props onto the matching DOM elements:

```tsx
import { useAgendaView } from '@big-calendar/react'

function MyAgendaView() {
  const { rows, components, messages, root, header, body, getRowProps } =
    useAgendaView()

  // Return nothing when a different view is active.
  if (rows === null) return null

  const { DateSlot, EventSlot, EmptySlot } = components

  return (
    <div {...root}>
      <div {...header}>
        <span>{messages.date}</span>
        <span>{messages.time}</span>
        <span>{messages.event}</span>
      </div>
      {rows.length === 0 ? (
        <EmptySlot message={messages.noEventsInRange} />
      ) : (
        <div {...body}>
          {rows.map((row) => (
            <div key={row.day} {...getRowProps(row)}>
              <DateSlot day={row.day} label={row.label} />
              {row.events.map((item) => (
                <EventSlot
                  key={item.key}
                  event={item.event}
                  title={item.title}
                  time={item.time}
                  allDay={item.allDay}
                />
              ))}
            </div>
          ))}
        </div>
      )}
    </div>
  )
}
```

Then register your view and pass the component to `CalendarProvider`:

```tsx
<CalendarProvider
  views={[Views.AGENDA]}
  defaultView={Views.AGENDA}
>
  <MyAgendaView />
</CalendarProvider>
```

## When to use this hook vs. `AgendaView`

Use `AgendaView` when the default rendering is good enough. Use `useAgendaView`
when you need to change the HTML structure of the view itself — for example,
to render events in a table instead of a CSS grid, or to add custom wrapper
elements. For changing individual cells or labels, override the `components.agenda`
slots instead (no hook required).

# Geometry Styles

The geometry style functions translate the numeric layout values produced by the view
model into plain `CSSProperties` objects you can spread onto a `style={}` prop. They
are the bridge between the calendar's internal coordinate system and the CSS that
positions every event, segment, slot, and indicator on screen.

You only need these when building a **fully custom headless view** (see
[Headless API](?path=/docs/advanced-headless-api--docs)). The built-in view components
call them internally, so nothing here is required for slot overrides.

---

## How they work

Every function wraps a corresponding utility in `@big-calendar/core` that emits CSS
custom-property assignments (e.g. `--bc-event-top: 25%`). The React wrappers here
narrow that output to `CSSProperties` so TypeScript is happy with `style={eventBoxStyle(g)}`.

---

## Event positioning

### `eventBoxStyle(geometry)`

Positions a timed event box inside a day column. Spread the result onto the outermost
element of a timed event.

```tsx
import { eventBoxStyle } from '@big-calendar/react'

<div
  className="bc-event"
  style={eventBoxStyle({ top, height, left, width, zIndex })}
>
  ...
</div>
```

**`EventBoxGeometry` properties**

<table>
  <thead><tr><th>Property</th><th>Type</th><th>Description</th></tr></thead>
  <tbody>
    <tr><td><code>top</code></td><td><code>number</code></td><td>Distance from the column top, as a percentage of total column height.</td></tr>
    <tr><td><code>height</code></td><td><code>number</code></td><td>Event height, as a percentage of total column height.</td></tr>
    <tr><td><code>left</code></td><td><code>number</code></td><td>Horizontal offset within the column's overlap group (0–1).</td></tr>
    <tr><td><code>width</code></td><td><code>number</code></td><td>Horizontal span within the column's overlap group (0–1).</td></tr>
    <tr><td><code>zIndex</code></td><td><code>number</code></td><td>Stack order for events that share vertical space.</td></tr>
  </tbody>
</table>

### `segmentStyle(geometry)`

Positions a month-view event segment (or an all-day segment) inside a grid row.
Spread the result onto the outermost element of the segment.

```tsx
import { segmentStyle } from '@big-calendar/react'

<div
  className="bc-segment"
  style={segmentStyle({ left, span, row })}
>
  ...
</div>
```

**`SegmentGeometry` properties**

<table>
  <thead><tr><th>Property</th><th>Type</th><th>Description</th></tr></thead>
  <tbody>
    <tr><td><code>left</code></td><td><code>number</code></td><td>1-based start column in the week row.</td></tr>
    <tr><td><code>span</code></td><td><code>number</code></td><td>Number of columns the segment spans.</td></tr>
    <tr><td><code>row</code></td><td><code>number</code></td><td>1-based stack level (first event is row 1).</td></tr>
  </tbody>
</table>

## Grid sizing

### `monthGridStyle(columnCount)`

Sets the number of day columns in a month grid. Call once on the grid container.

```tsx
import { monthGridStyle } from '@big-calendar/react'

<div className="bc-month-grid" style={monthGridStyle(7)}>
  ...
</div>
```

**`columnCount`** — number of columns (almost always `7` for a standard week).

---

### `dayCountStyle(dayCount)`

Sets the number of day columns in a time-grid header or body. Call once on the header
or body root.

```tsx
import { dayCountStyle } from '@big-calendar/react'

<div className="bc-time-header" style={dayCountStyle(5)}>
  ...
</div>
```

**`dayCount`** — number of visible day columns (5 for work-week, 7 for week, 1 for day).

---

### `slotCountStyle(slotCount)`

Sets the number of time-slot rows in a single day column.

```tsx
import { slotCountStyle } from '@big-calendar/react'

<div className="bc-day-column" style={slotCountStyle(96)}>
  {/* 96 slots = 15-min intervals across 24 hours */}
</div>
```

**`slotCount`** — total number of minor slots in the column.

---

### `slotGroupStyle(groupSize)`

Groups minor time slots under one major label in the gutter. The gutter label spans
`groupSize` minor slots.

```tsx
import { slotGroupStyle } from '@big-calendar/react'

<div className="bc-slot-group" style={slotGroupStyle(4)}>
  {/* 4 minor slots per hour label */}
</div>
```

**`groupSize`** — number of minor slots per labelled group (e.g. `4` for 15-min slots
under hourly labels).

---

### `agendaRowsStyle(rowCount)`

Sets the number of agenda day-group rows in the agenda grid.

```tsx
import { agendaRowsStyle } from '@big-calendar/react'

<div className="bc-agenda" style={agendaRowsStyle(rows.length)}>
  ...
</div>
```

**`rowCount`** — number of day groups currently rendered.

---

## Indicators

### `nowIndicatorStyle(offset)`

Positions the "now" line within a day column as a percentage of the column height.

```tsx
import { nowIndicatorStyle } from '@big-calendar/react'

<div className="bc-now-indicator" style={nowIndicatorStyle(offset)} />
```

**`offset`** — fractional position from the top (0 = midnight, 0.5 = noon, 1 = end of day).

---

### `selectionStyle(geometry)`

Positions the visual selection highlight inside a day column during an in-progress
slot selection.

```tsx
import { selectionStyle } from '@big-calendar/react'

<div className="bc-selection" style={selectionStyle({ top, height })} />
```

**`SelectionGeometry` properties**

<table>
  <thead><tr><th>Property</th><th>Type</th><th>Description</th></tr></thead>
  <tbody>
    <tr><td><code>top</code></td><td><code>number</code></td><td>Distance from the column top, as a percentage of total column height.</td></tr>
    <tr><td><code>height</code></td><td><code>number</code></td><td>Selection height, as a percentage of total column height.</td></tr>
  </tbody>
</table>

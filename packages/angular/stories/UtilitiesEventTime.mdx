# Event Time Utilities

Two utilities handle the two most common cross-cutting concerns in custom event rendering: formatting a start–end time as a human-readable string, and computing where a floating element (popover, tooltip, dialog) should appear on screen.

Both are available from `@big-calendar/core/utils`.

```ts
import { formatEventTime, positionFloating } from '@big-calendar/core/utils'
```

---

## `formatEventTime`

Formats an event's start and end into a localizer-aware display string. The default event components use this to build the time label shown inside an event box or agenda row.

When building a **fully custom** component that needs to format time itself — for example, a custom agenda event that displays start and end on separate lines — call `formatEventTime` directly:

```ts
import { Component, inject } from '@angular/core'
import { formatEventTime } from '@big-calendar/core/utils'
import { injectCalendar } from '@big-calendar/angular'

@Component({
  standalone: true,
  selector: 'my-agenda-event',
  template: `<span>{{ label }}</span>`,
  inputs: ['event'],
})
export class MyAgendaEventComponent {
  event: any

  private readonly ctx = injectCalendar()

  get label(): string {
    const store = this.ctx.storeSignal()
    if (!store) return ''
    const start = store.accessors.start(this.event)
    const end   = store.accessors.end(this.event)
    return formatEventTime({
      localizer:   store.localizer,
      allDayLabel: store.messages.allDay,
      start:       typeof start === 'string' ? start : null,
      end:         typeof end   === 'string' ? end   : null,
      allDay:      store.accessors.allDay(this.event) ?? false,
    })
  }
}
```

### `FormatEventTimeArgs`

<table>
<thead><tr><th>Property</th><th>Type</th><th>Required</th><th>Description</th></tr></thead>
<tbody>
<tr><td><code>localizer</code></td><td><code>LocalizerContract</code></td><td>yes</td><td>The active localizer (read from the calendar store).</td></tr>
<tr><td><code>allDayLabel</code></td><td><code>string</code></td><td>yes</td><td>The "all day" label string (read from <code>store.messages.allDay</code>).</td></tr>
<tr><td><code>start</code></td><td><code>string | null</code></td><td>yes</td><td>Resolved event start in RFC 3339/9557 format, or <code>null</code> if unresolved.</td></tr>
<tr><td><code>end</code></td><td><code>string | null</code></td><td>yes</td><td>Resolved event end, or <code>null</code> if unresolved. When <code>null</code>, only the start is formatted.</td></tr>
<tr><td><code>allDay</code></td><td><code>boolean</code></td><td>yes</td><td>When <code>true</code>, returns <code>allDayLabel</code> regardless of <code>start</code> / <code>end</code>.</td></tr>
<tr><td><code>format</code></td><td><code>'agendaTime' | 'time'</code></td><td>no</td><td>Format role passed to the localizer. Defaults to <code>'agendaTime'</code>.</td></tr>
</tbody>
</table>

**Returns** `string` — either `allDayLabel`, a single formatted time, or a formatted range "start – end".

## `positionFloating`

Computes a `position: fixed` coordinate for a top-layer element (native Popover API or modal `<dialog>`) relative to an anchor element. Used internally by the Popover and Tooltip components; exported so you can position custom top-layer surfaces using the same engine.

```ts
import { positionFloating } from '@big-calendar/core/utils'

const { x, y, placement } = await positionFloating(anchorEl, floatingEl, {
  placement: 'bottom-start',
  offset: 8,
})

floatingEl.style.position         = 'fixed'
floatingEl.style.insetInlineStart = `${x}px`
floatingEl.style.insetBlockStart  = `${y}px`
```

The function is async because it lazily imports `@floating-ui/core` on first call (keeping it out of the initial JS bundle). Subsequent calls reuse the cached module.

### Signature

```ts
positionFloating(
  reference: HTMLElement,
  floating:  HTMLElement,
  options?:  PositionFloatingOptions,
): Promise<FloatingPosition>
```

### `PositionFloatingOptions`

<tr><td><code>offset</code></td><td><code>number</code></td><td><code>4</code></td><td>Gap between the anchor and the floating element, in pixels.</td></tr>

### `FloatingPosition`

<table>
<thead><tr><th>Property</th><th>Type</th><th>Description</th></tr></thead>
<tbody>
<tr><td><code>x</code></td><td><code>number</code></td><td>Distance from the inline-start viewport edge, in pixels.</td></tr>
<tr><td><code>y</code></td><td><code>number</code></td><td>Distance from the block-start viewport edge, in pixels.</td></tr>
<tr><td><code>placement</code></td><td><code>FloatingPlacement</code></td><td>The placement that was actually used (may differ from the requested one if flipped).</td></tr>

### `FloatingPlacement`

A floating-ui placement string. Common values: `'bottom-start'`, `'bottom-end'`, `'top-start'`, `'top-end'`, `'right'`, `'left'`. The middleware will flip to the opposite side if the preferred side clips the viewport.

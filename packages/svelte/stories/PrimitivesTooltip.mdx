# Tooltip

A lightweight hover tooltip built on the same top-layer positioning stack as `<Popover>`. It appears on mouse-over and disappears when the pointer leaves.

## Where it is used

Big Calendar does not attach tooltips to events automatically. `<Tooltip>` is exported as a building block for custom event slot components that need hover text — for example, a truncated event title with a full-text tooltip, or a hover card with extended event details.

## API

```ts
import { Tooltip } from '@big-calendar/svelte'
```

In Svelte, `<Tooltip>` wraps its trigger via the `children` snippet:

```svelte
<!-- TooltipEvent.svelte -->
<script lang="ts">
  import { Tooltip } from '@big-calendar/svelte'
  let { event, title }: { event: { location?: string }; title: string } = $props()
</script>

<Tooltip label={`${title} — ${event.location}`} placement="top">
  <span class="my-event-label">{title}</span>
</Tooltip>
```

> **Svelte note.** Vue's `content` prop is called **`label`** in the Svelte adapter. The children snippet is the trigger element — any element that accepts mouse-over events works.

## Styling

The base class is `bc-tooltip`. It uses a dark background and light text by default:

```css
.bc-tooltip {
  background: var(--bc-color-fg);
  color: var(--bc-color-bg);
  border-radius: var(--bc-radius-sm);
  font-size: var(--bc-font-size-sm);
}
```

Override via CSS custom properties or target `.bc-tooltip` directly.

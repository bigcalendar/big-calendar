# Headless API

The `<Calendar>` component and the `<MonthView>`, `<TimeGridView>`, `<AgendaView>` components are the default path — they render a complete, styled calendar using the library's HTML structure. The **headless API** gives you the view model and all element props so you can build exactly the markup you need, while the library still handles all the logic.

## When to use the composables

Use the view components when the default rendering is good enough, or when you only need to swap individual slots via `components.month`, `components.time`, or `components.agenda`.

Use the composables when you need to change the **HTML structure** of a view — for example, rendering the month grid as a `<table>`, adding custom wrapper elements between the slots, or building a completely custom layout.

## The pattern

Every composable follows the same pattern:

1. Call the composable inside a `.svelte` component that lives inside a `CalendarProvider`.
2. Guard on the returned `grid.current` (or `rows.current`) — it is `null` when a different view is active.
3. Spread the returned element-prop objects onto your DOM elements.
4. Use the returned getter functions to build per-item props.

```svelte
<!-- MyMonthView.svelte -->
<script>
  import { useMonthView, EventButton } from '@big-calendar/svelte'

  let rootEl = $state(null)
  let gridEl = $state(null)

  const { grid, components, getRootProps, getMonthGridProps, getDaySlotProps, getSegmentProps } =
    useMonthView(
      () => rootEl,
      () => gridEl,
    )

  const { Weekday, DateCell, EventSlot } = components
</script>

{#if grid.current !== null}
  <div bind:this={rootEl} {...getRootProps()}>
    <!-- your custom HTML structure -->
  </div>
{/if}
```

Place your component inside `CalendarProvider` alongside the toolbar:

```svelte
<CalendarProvider {events} defaultView="month">
  <Toolbar />
  <div class="bc-calendar">
    <MyMonthView />
  </div>
</CalendarProvider>
```

> **Svelte key differences from Vue:**
> - `useMonthView` takes **two getter functions** as arguments: one for the root element, one for the grid element. These replace Vue's internal `ref` management.
> - `grid` and `rows` are reactive objects with a `.current` property — guard on `grid.current !== null`, not `grid !== null`.
> - `getRootProps()` and `getMonthGridProps()` are **functions** — call them at render time. In Vue these were plain objects.
> - Spread element props with `{...getRootProps()}` (Svelte's JSX-like spread) instead of Vue's `v-bind`.
> - Bind element refs with `bind:this={el}` instead of Vue's `:ref`.

## Element-spread props

The composables return prop-getter functions like `getRootProps()`, `getMonthGridProps()`, `getBody()`, etc. Each returns a plain object of HTML attributes — `class`, `aria-*`, event handlers — designed to be spread directly onto a DOM element:

```svelte
<div bind:this={rootEl} {...getRootProps()}>
  <div bind:this={gridEl} {...getMonthGridProps()}>
    ...
  </div>
</div>
```

Do not destructure or pick from these objects individually — spread the whole thing so no attributes are dropped.

## Slot components from the composable

Each composable returns a `components` object with the resolved slot components — either the defaults or the `components.month` / `components.time` / `components.agenda` overrides you passed to `CalendarProvider`. Use these to render individual pieces:

```svelte
const { Weekday, DateCell, EventSlot, ShowMore } = components
```

This ensures your custom layout respects the `components` overrides just like the default view does. Use `<svelte:component this={...}>` to render them:

```svelte
<svelte:component this={EventSlot} event={seg.event} title={seg.title} />
```

## EventButton

When rendering event segments yourself, wrap them in `<EventButton>` and spread the props returned by the composable's getter (e.g. `getSegmentProps`, `getEventProps`). `EventButton` is what wires up click, keyboard, and drag-and-drop interactions for an event:

```svelte
<script>
  import { EventButton } from '@big-calendar/svelte'
</script>

{#each week.segments as seg (seg.key)}
  <EventButton {...getSegmentProps(seg)}>
    <svelte:component this={EventSlot} event={seg.event} title={seg.title} />
  </EventButton>
{/each}
```

# Custom Components

Every visual piece of the calendar is replaceable via the `components` prop on `CalendarProvider`. In Svelte, a "component" is any `.svelte` file — pass the imported component class directly. No special base class or wrapper is required.

More specific (per-view) slots always take precedence over shared ones. You can override as many or as few as you like; unset slots use the built-in defaults.

```svelte
<!-- App.svelte -->
<script>
  import { CalendarProvider, Calendar } from '@big-calendar/svelte'
  import MyToolbar from './MyToolbar.svelte'
  import MyMonthEvent from './MyMonthEvent.svelte'
  import MyTimeEvent from './MyTimeEvent.svelte'
</script>

<CalendarProvider
  components={{
    toolbar: MyToolbar,
    month: { event: MyMonthEvent },
    time:  { event: MyTimeEvent },
  }}
>
  <Calendar />
</CalendarProvider>
```

---

## Svelte vs React

In React, a slot component is a function. In Svelte, it is a **Svelte component** — a `.svelte` file imported and passed as a class reference. The pattern is:

```svelte
<!-- BoldWeekday.svelte — a slot component -->
<script lang="ts">
  let { long, day, short }: { long: string; day: string; short: string } = $props()
</script>
<strong style="text-transform:uppercase">{long}</strong>
```

```svelte
<!-- App.svelte — pass it via components -->
<script>
  import { CalendarProvider, Calendar } from '@big-calendar/svelte'
  import BoldWeekday from './BoldWeekday.svelte'
</script>

<CalendarProvider components={{ month: { weekday: BoldWeekday } }}>
  <Calendar />
</CalendarProvider>
```

The `eventSlot` prop in `MonthShowMoreProps` and `TimeShowMoreProps` is a Svelte component class. Render it with `<svelte:component this={eventSlot} event={item.event} title={item.title} />`.

### `toolbar`

Replaces the navigation toolbar. See **Calendar → Toolbar** for the full props reference and usage example.

## Month view slots (`components.month`)

### `weekday`

A column heading in the weekday header row (Sun, Mon, Tue, …).

```svelte
<!-- MyWeekday.svelte -->
<script lang="ts">
  let { day, long, short }: { day: string; long: string; short: string } = $props()
</script>
<abbr title={long}>{short}</abbr>
```

### `dateCell`

The date header inside a day cell — the number, today/off-range state, and drill-down handler.

```svelte
<!-- MyDateCell.svelte -->
<script lang="ts">
  let { label, isToday, isOffRange, onDrillDown, day }: {
    label: string; isToday: boolean; isOffRange: boolean
    onDrillDown: () => void; day: string
  } = $props()
</script>
<button type="button" onclick={onDrillDown} aria-label={`Drill down to ${label}`}>
  {label}
</button>
```

### `showMore`

The "+N more" overflow indicator shown when a week row exceeds `weekEventLimit`. The `eventSlot` prop is a Svelte component class — use it to render each hidden event consistently.

```svelte
<!-- MyShowMore.svelte -->
<script lang="ts">
  import type { Component } from 'svelte'
  import type { ShowMoreEvent } from '@big-calendar/svelte'

  let { count, label, day, events, eventSlot: EventSlot }: {
    count: number; label: string; day: string
    events: ReadonlyArray<ShowMoreEvent>; eventSlot: Component
  } = $props()
</script>
<details>
  <summary>{label}</summary>
  {#each events as item (item.key)}
    <svelte:component this={EventSlot} event={item.event} title={item.title} />
  {/each}
</details>
```

  <thead><tr><th>Prop</th><th>Type</th><th>Description</th></tr></thead>
  <tbody>
    <tr><td><code>count</code></td><td><code>number</code></td><td>How many events overflowed.</td></tr>
    <tr><td><code>label</code></td><td><code>string</code></td><td>Localized label ("+2 more").</td></tr>
    <tr><td><code>day</code></td><td><code>string</code></td><td>Day-start string of the overflow anchor.</td></tr>
    <tr><td><code>events</code></td><td><code>ReadonlyArray&lt;ShowMoreEvent&lt;TEvent&gt;&gt;</code></td><td>The hidden events.</td></tr>
    <tr><td><code>eventSlot</code></td><td><code>Component</code></td><td>The configured month event slot — use it to render each hidden event consistently.</td></tr>

## See it in action

The **Calendar → Custom Rendering** stories each replace exactly one slot and show the live result. There is a story per slot, so you can jump straight to the one you want to customize.

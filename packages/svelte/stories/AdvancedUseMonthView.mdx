# useMonthView

A composable that composes all the logic needed to render a custom month grid. Call it inside a component that lives inside a `CalendarProvider`. See **Advanced → Headless API** for the general pattern.

```ts
import { useMonthView } from '@big-calendar/svelte'
```

## Signature

```ts
useMonthView<TEvent = unknown>(
  getRootEl: () => HTMLElement | null,
  getGridEl: () => HTMLElement | null,
): UseMonthViewReturn<TEvent>
```

`useMonthView` takes **two getter functions** that return the root and grid DOM elements. Use `$state` variables with `bind:this` to supply them.

## Usage example

```svelte
<!-- MyMonthView.svelte -->
<script>
  import { EventButton, useMonthView } from '@big-calendar/svelte'

  let rootEl = $state(null)
  let gridEl = $state(null)

  const {
    grid, components, getMessages, announcement, announcer,
    getRootProps, getMonthGridProps, monthHeader, weekRow,
    slotsContainer, backgroundsContainer, eventsContainer,
    getDaySlotProps, getWeekSelectionBand, getWeekPreviewBand,
    getSegmentProps, getShowMoreCellProps,
  } = useMonthView(
    () => rootEl,
    () => gridEl,
  )

  const { Weekday, DateCell, EventSlot, ShowMore } = components
</script>

{#if grid.current !== null}
  <div bind:this={rootEl} {...getRootProps()}>
    <div {...announcer}>{announcement()}</div>

    <div {...monthHeader}>
      {#each grid.current.weekdays as wd (wd.day)}
        <svelte:component this={Weekday} day={wd.day} long={wd.long} short={wd.short} />
      {/each}
    </div>

    <div bind:this={gridEl} {...getMonthGridProps()}>
      {#each grid.current.weeks as week, wi (week.key)}
        <div {...weekRow}>
          <div {...slotsContainer}>
            {#each week.days as cell, di (cell.day)}
              <div {...getDaySlotProps(cell, wi, di)} />
            {/each}
          </div>

          <div {...backgroundsContainer}>
            {#each week.days as cell, di (cell.day)}
              <div>
                <svelte:component this={DateCell} {...cell} onDrillDown={() => {}} />

                {#if getShowMoreCellProps(cell, di, week.moreRow) as more}
                  <div {...more.wrapper}>
                    <svelte:component
                      this={ShowMore}
                      {...more.showMore}
                      label={getMessages().showMore(more.showMore.count)}
                      eventSlot={EventSlot}
                    />
                  </div>
                {/if}
              </div>
            {/each}
          </div>

          {#if getWeekSelectionBand(wi)}
            <div {...getWeekSelectionBand(wi)} />
          {/if}
          {#if getWeekPreviewBand(week)}
            <div {...getWeekPreviewBand(week)} />
          {/if}

          <div {...eventsContainer}>
            {#each week.segments as seg (seg.key)}
              <EventButton {...getSegmentProps(seg)}>
                <svelte:component this={EventSlot} event={seg.event} title={seg.title} />
              </EventButton>
            {/each}
          </div>
        </div>
      {/each}
    </div>
  </div>
{/if}
```

> **Key Svelte differences from Vue:**
> - `grid.current` not `grid` — always access the data through `.current`.
> - `getRootProps()` and `getMonthGridProps()` are called **as functions** at render time.
> - `announcement()` is a function — call it in the template.
> - Element refs use `bind:this={el}` and getter functions `() => el` passed to `useMonthView`.
> - Slot components render with `<svelte:component this={...} />` instead of `<component :is="...">`.
> - `{...obj}` spreads props; no `v-bind` needed.

## When to use this vs. `<MonthView>`

Use `<MonthView>` when the default rendering is sufficient. Use `useMonthView` when you need to change the HTML structure — render the grid as a `<table>`, add custom row wrappers, or change how segments are positioned.

For changing individual cells or labels without restructuring the HTML, use `components.month` slot overrides on `CalendarProvider` instead — no composable required.

# useAgendaView

A composable that composes all the data and element props needed to render a custom agenda view. Call it inside a component that lives inside a `CalendarProvider`. See **Advanced → Headless API** for the general pattern.

```ts
import { useAgendaView } from '@big-calendar/svelte'
```

## Signature

```ts
useAgendaView<TEvent = unknown>(): UseAgendaViewReturn<TEvent>
```

`useAgendaView` takes no arguments. Unlike the month and time-grid composables, no element getter functions are required.

## Usage example

```svelte
<!-- MyAgendaView.svelte -->
<script>
  import { useAgendaView } from '@big-calendar/svelte'

  const { rows, components, messages, root, header, headingCell, body, getRowProps } =
    useAgendaView()

  const { DateSlot, EventSlot, EmptySlot } = components
</script>

{#if rows.current !== null}
  <div {...root}>
    <div {...header}>
      <span {...headingCell}>{messages.date}</span>
      <span {...headingCell}>{messages.time}</span>
      <span {...headingCell}>{messages.event}</span>
    </div>

    {#if rows.current.length === 0}
      <svelte:component this={EmptySlot} message={messages.noEventsInRange} />
    {:else}
      <div {...body}>
        {#each rows.current as row (row.day)}
          <div {...getRowProps(row)}>
            <svelte:component this={DateSlot} day={row.day} label={row.label} />
            {#each row.events as item (item.key)}
              <svelte:component
                this={EventSlot}
                event={item.event}
                title={item.title}
                time={item.time}
                allDay={item.allDay}
              />
            {/each}
          </div>
        {/each}
      </div>
    {/if}
  </div>
{/if}
```

Place your component inside `CalendarProvider`:

```svelte
<CalendarProvider defaultView="agenda" {events}>
  <Toolbar />
  <div class="bc-calendar">
    <MyAgendaView />
  </div>
</CalendarProvider>
```

> **Key Svelte differences from Vue:**
> - `rows.current` not `rows` — always access the data through `.current`.
> - `root`, `header`, `headingCell`, and `body` are **static objects** — spread them with `{...obj}` at any point.
> - No element getter functions needed — `useAgendaView()` manages its own positioning via the calendar store.
> - `<svelte:component this={...} />` instead of `<component :is="...">`.

## When to use this vs. `<AgendaView>`

Use `<AgendaView>` when the default rendering is sufficient. Use `useAgendaView` when you need to change the HTML structure — render events in a `<table>`, add custom wrapper elements, or change how the date and event columns are laid out.

For changing individual cells or labels without restructuring, use `components.agenda` slot overrides on `CalendarProvider` — no composable required.

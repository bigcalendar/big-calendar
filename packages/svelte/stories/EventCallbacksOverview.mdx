# Event Callbacks

Big Calendar fires callbacks when a user interacts with an event block — clicking, double-clicking, right-clicking, or middle-clicking. These fire on **any** view and do **not** require `selectable={true}` to be enabled. They are completely independent of slot selection.

## The callbacks

### `onEventClick`

Fires when the user **single-clicks** an event. Receives the raw event object from your `events` array — the exact same value you passed in, with no wrapping.

```svelte
<CalendarProvider
  onEventClick={(event) => {
    editingEvent = event
    drawerOpen = true
  }}
>
```

### `onEventDoubleClick`

Same as `onEventClick`, fires on a **double-click**.

```svelte
<CalendarProvider
  onEventDoubleClick={(event) => openFullEditor(event)}
>
```

### `onEventRightClick`

Fires on a **right-click**. Receives a single named-args object with `event` (your event object) and `domEvent` (the browser `MouseEvent`). The browser's native context menu still appears unless you call `domEvent.preventDefault()`.

```svelte
<CalendarProvider
  onEventRightClick={({ event, domEvent }) => {
    domEvent.preventDefault()  // suppress the browser context menu
    contextMenu = { event, x: domEvent.clientX, y: domEvent.clientY }
  }}
>
```

### `onEventMiddleClick`

Fires on a **middle-click** (scroll-wheel button press). Same named-args shape as `onEventRightClick`. Useful for opening an event in a new tab without disrupting the calendar.

```svelte
<CalendarProvider
  onEventMiddleClick={({ event, domEvent }) => {
    domEvent.preventDefault()
    window.open(`/events/${event.id}`, '_blank')
  }}
>
```

## Combining with slot selection

Event callbacks and slot selection coexist without any conflict. Add `selectable={true}` to `CalendarProvider` to enable slot callbacks alongside event callbacks:

```svelte
<CalendarProvider
  selectable={true}
  onSlotSelect={({ start, end }) => openCreateDialog({ start, end })}
  onEventClick={(event) => openEditDialog(event)}
>
```

See the **Selection → Overview** for the full slot callback reference.

## See it in action

The **Events → Event Callbacks** story has all four callbacks wired to the **Actions** panel. Click any event to see the full event object logged. Use the **Controls** panel to switch views.

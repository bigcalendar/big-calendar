# @big-calendar/styles

The style layer for Big Calendar. It ships a single CSS file (or granular
pieces you can pick from) that handles the reset, design tokens, layout, and
visual skin for every built-in component. The React package reads these class
names — it ships no CSS of its own.

## Installing

```bash
yarn add @big-calendar/styles
```

Then import the full bundle once, near the top of your app:

```ts
import '@big-calendar/styles/index.css'
```

Or pick only what you need:

```ts
import '@big-calendar/styles/reset.css'
import '@big-calendar/styles/tokens.css'
import '@big-calendar/styles/layout.css'
import '@big-calendar/styles/components/toolbar.css'
import '@big-calendar/styles/components/month.css'
import '@big-calendar/styles/components/timegrid.css'
import '@big-calendar/styles/components/event.css'
import '@big-calendar/styles/components/agenda.css'
import '@big-calendar/styles/components/popover.css'
import '@big-calendar/styles/components/selection.css'
```

## Cascade layers

CSS cascade layers let the browser know which styles are "base" (the library)
and which are "yours" (the override). When two rules have the same specificity,
the layer order decides which wins — layers declared later win. Big Calendar
puts its styles in named layers so that your overrides *always* win, even if
your selector is simple.

The layer order is declared once at the top of `index.css`:

```css
@layer bc.reset, bc.tokens, bc.layout, bc.components, bc.theme, bc.overrides;
```

<table>
  <thead>
    <tr><th>Layer</th><th>What lives there</th></tr>
  </thead>
  <tbody>
    <tr><td><code>bc.reset</code></td><td>Scoped <code>box-sizing</code>, button/list/input resets, <code>font-family</code>/<code>color</code> base</td></tr>
    <tr><td><code>bc.tokens</code></td><td>All <code>--bc-*</code> CSS custom property defaults (design tokens)</td></tr>
    <tr><td><code>bc.layout</code></td><td>Grid/flex structural rules — no colors, just geometry</td></tr>
    <tr><td><code>bc.components</code></td><td>Visual skin: colors, borders, hover states, typography</td></tr>
    <tr><td><code>bc.theme</code></td><td>Reserved for a future official theme file (empty today)</td></tr>
    <tr><td><code>bc.overrides</code></td><td>The slot for your own overrides — target it to always win</td></tr>
  </tbody>
</table>

To override any rule without fighting specificity, put your CSS in
`bc.overrides`:

```css
@layer bc.overrides {
  .bc-toolbar {
    padding: 1rem 1.5rem;
  }
}
```

Or put your styles **outside** all layers entirely — unlayered styles always
beat layered ones, regardless of order:

```css
/* This wins over everything in any bc.* layer */
.bc-toolbar {
  padding: 1rem 1.5rem;
}
```

## Design tokens

Every visual value is a CSS custom property, prefixed `--bc-*`. They are
defined on `:where(.bc-calendar)` at zero specificity, so any rule that targets
`.bc-calendar` directly overrides them:

```css
.bc-calendar {
  --bc-color-event-bg: #8b5cf6; /* your purple brand color */
}
```

## Class reference

All classes are in the `bc-` namespace. They are added by the framework
adapter (the React package); you do not write them yourself. They are documented
here so you can target them in your own CSS.

### Layout root

```
.bc-calendar          ← the sizing container; tokens and container queries root
  .bc-viewport        ← inner scroll boundary; overflow: hidden
```

`.bc-calendar` must receive an explicit height from its parent. The React
adapter renders it inside whatever element you give it.

`.bc-toolbar` can appear **inside** or **outside** `.bc-calendar` — it gets the
reset and base font either way.

### Toolbar

```
.bc-toolbar           ← flex row; nav + label + view switcher
  .bc-toolbar-group   ← flex group (Back/Forward buttons live here)
  .bc-toolbar-label   ← the current range label ("June 2025")
  button              ← navigation and view-switch buttons
    [aria-pressed="true"] or .bc-active  ← the active view button
```

### Month view

```
.bc-month             ← grid; header row + body rows
  .bc-month-header    ← the "Sun Mon Tue …" weekday names row
    .bc-weekday       ← one weekday name cell
      .bc-weekday-long   ← full name ("Sunday"); hidden below 34 rem
      .bc-weekday-short  ← abbreviated ("Su"); hidden above 34 rem
  .bc-month-grid      ← 7-column grid of week rows
    .bc-month-week    ← one week row; subgrid so segments share column tracks
      .bc-week-backgrounds    ← date cells layer (pointer-events: none)
        .bc-date-cell         ← one day cell
          .bc-date-number     ← the date number label (re-enables pointer-events)
          [.bc-today]         ← modifier: today's cell
          [.bc-off-range]     ← modifier: day outside the displayed month
      .bc-month-slots         ← transparent hit-target layer for slot selection
        .bc-month-slot        ← one per day; keyboard-focusable
      .bc-week-events         ← segment rows layer
        .bc-segment           ← one event segment (can span multiple day columns)
          [.bc-event-draggable]  ← DnD is active for this segment
          [.bc-selected] / [aria-selected="true"]  ← selected state
          .bc-event-title     ← event title text
          .bc-resize-handle.bc-resize-handle-start   ← left resize strip
          .bc-resize-handle.bc-resize-handle-end     ← right resize strip
        .bc-show-more-cell    ← overflow placeholder when events don't fit
          .bc-show-more       ← the "+ N more" button
```

### Time-grid view (day / week / work-week)

```
.bc-time-grid                 ← 2-column grid (gutter | columns)
  .bc-time-header             ← day heading row
    .bc-time-header-gutter    ← top-left spacer cell
    .bc-day-heading           ← one day name cell
      [.bc-today]             ← modifier: today's heading
  .bc-allday-row              ← all-day band above the scrollable slots
    .bc-allday-label          ← "all-day" text label
    .bc-allday-slots          ← transparent hit-targets (one per day)
      .bc-allday-slot         ← one per day; keyboard-focusable
        [.bc-today]           ← today highlight
    .bc-allday-selection      ← overlay grid for the selection band
    .bc-allday-segments       ← stacked all-day event segments (same as month)
  .bc-time-body               ← scrollable slot area
    .bc-time-gutter           ← time label column
      .bc-time-label          ← one hour label
    .bc-day-column            ← one day column; positioning context for events
      [.bc-today]             ← today highlight
      .bc-time-slots          ← absolute grid of per-slot hit targets
        .bc-time-slot         ← one slot cell; keyboard-focusable
      .bc-event               ← one timed event box
        [.bc-event-draggable]
        [.bc-selected] / [aria-selected="true"]
        [.bc-event-grabbed]   ← keyboard drag: event is "in hand"
        .bc-event-title
        .bc-event-time
        .bc-resize-handle.bc-resize-handle-start   ← top resize strip
        .bc-resize-handle.bc-resize-handle-end     ← bottom resize strip
      .bc-bg-event            ← background event (no pointer events)
      .bc-now-indicator       ← current-time line
```

### Resource time-grid

When you pass a `resources` array to `CalendarProvider`, the adapter wraps the
time grid in additional containers and adds tier headers:

```
.bc-time-grid-resources          ← 2D scroll container; one column per resource×day
  [.bc-time-grid-resources-day]  ← modifier: day view (wider columns)
  .bc-time-head                  ← sticky header block (stays put on vertical scroll)
    .bc-time-header              ← day-name or resource-title row
      .bc-time-header-gutter     ← frozen top-left corner
      .bc-resource-header        ← one resource title cell
      .bc-day-heading            ← one day name (tiered week header only)
    .bc-time-header-tiered       ← modifier: two-tier (resource name over day names)
      .bc-resource-day-head      ← day-name subgrid cell inside the tier
    .bc-allday-row               ← all-day band (inside the sticky head)
      .bc-allday-resource        ← one resource's all-day lane
        .bc-allday-slot          ← hit target (absolute, fills the lane)
        .bc-allday-resource-stack   ← stacked flow of all-day segments
      .bc-allday-resource-week   ← week variant: day sub-grid inside the lane
        .bc-allday-resource-slots     ← transparent per-day hit targets
        .bc-allday-resource-segments  ← per-day segment grid
  .bc-time-body                  ← slot body (non-scrolling inside the container)
    .bc-time-gutter              ← frozen inline-start time labels
    .bc-day-column               ← one day/resource column (events positioned here)
```

### Agenda view

.bc-agenda                ← 3-column grid (date | time | event)

### Target a specific view

All view roots are direct children of `.bc-viewport`, so you can scope to one
view at a time:

```css
@layer bc.overrides {
  /* Agenda only: bigger date column */
  .bc-agenda-date {
    min-inline-size: 8rem;
  }

  /* Month only: larger weekday names */
  .bc-month-header .bc-weekday {
    font-size: var(--bc-font-size-lg);
  }
}
```

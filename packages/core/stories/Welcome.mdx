# @big-calendar/core

The framework-agnostic calendar engine. It owns all the date math, view models,
event layout, and the reactive store — and renders **nothing**. UI lives in the
thin per-framework packages (for example `@big-calendar/react`).

Because core is headless, it has no components to show in Storybook. This page is
a placeholder; the engine's public API (exports, the `Views` / `Navigate`
constants, `createCalendarStore`, and the localizer contract) will be documented
here in a later pass.

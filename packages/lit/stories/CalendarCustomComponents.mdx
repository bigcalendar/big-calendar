# Custom Components

Because `@big-calendar/lit` uses the light DOM, the most direct way to customise the calendar's appearance is **plain CSS**. Every element the calendar renders carries a `.bc-*` class that you can target from any stylesheet on the page — no `::part()` or `::slotted()` workarounds needed.

See the **Calendar → Custom Rendering** stories for live examples of each CSS customisation point.

---

## CSS customisation

```css
/* Make weekday column headers bold purple */
.bc-weekday {
  font-weight: 700;
  color: rebeccapurple;
}

/* Highlight today's date number with a filled circle */
.bc-today .bc-date-number {
  background: rebeccapurple;
  color: white;
  border-radius: 50%;
}

/* Add a coloured left border to agenda event cells */
.bc-agenda-event-cell {
  border-left: 3px solid rebeccapurple;
  padding-inline-start: 0.75rem;
}
```

## Extending view elements

For deeper customisation — changing DOM structure, adding custom event content, or injecting render logic — you can extend the view element classes and register your own custom element tag.

```ts
import { MonthViewElement } from '@big-calendar/lit'
import { customElement } from 'lit/decorators.js'

@customElement('my-month-view')
class MyMonthViewElement extends MonthViewElement {
  // Override methods or add properties here
}
```

Then use `<my-month-view>` instead of `<bc-month-view>` inside `<bc-calendar>`:

```ts
html`
  <bc-calendar .localizer=${localizer} .events=${events}>
    <bc-default-toolbar></bc-default-toolbar>
    <my-month-view></my-month-view>
    <bc-time-grid-view></bc-time-grid-view>
    <bc-agenda-view></bc-agenda-view>
  </bc-calendar>
`
```

> **Note:** Render function customisation (the `components` prop from the React/Vue/Angular adapters) is on the Lit roadmap. For the current release, CSS and class extension are the primary customisation surfaces.

## Injecting styles at runtime

When you need to add styles from JavaScript (for example, in a story or a component that manages its own lifecycle), create and remove a `<style>` element:

```ts
class MyCalendarComponent extends LitElement {
  private _styleEl: HTMLStyleElement | null = null

  override connectedCallback() {
    super.connectedCallback()
    this._styleEl = document.createElement('style')
    this._styleEl.textContent = `
      .bc-weekday { text-transform: uppercase; letter-spacing: 0.08em; }
    `
    document.head.appendChild(this._styleEl)
  }

  override disconnectedCallback() {
    super.disconnectedCallback()
    this._styleEl?.remove()
    this._styleEl = null
  }
}
```

For production use, add the rules to your global stylesheet instead.

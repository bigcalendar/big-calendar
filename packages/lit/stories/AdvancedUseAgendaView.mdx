# AgendaViewElement

`AgendaViewElement` (`<bc-agenda-view>`) renders the agenda (list) view. It consumes `calendarContext` via `@lit/context` and renders only when the active view is `'agenda'`.

## How it works internally

`AgendaViewElement` follows the same pattern as all Lit view elements:

1. It creates a `ContextConsumer` for `calendarContext`.
2. When context arrives, it subscribes to the store signals with `effect()` from `@preact/signals-core` and calls `requestUpdate()` on each change.
3. It computes the agenda row model from the store on every render.
4. It renders rows grouped by date, with each row containing event entries.

## Extending AgendaViewElement

Extend the class to add custom rendering or override specific methods:

```ts
import { AgendaViewElement } from '@big-calendar/lit'
import { customElement } from 'lit/decorators.js'
import { html } from 'lit'

@customElement('my-agenda-view')
class MyAgendaViewElement extends AgendaViewElement {
  // Override render to add custom markup around the default output
  override render() {
    return html`
      <div class="my-agenda-wrapper">
        ${super.render()}
      </div>
    `
  }
}
```

Then use `<my-agenda-view>` in place of `<bc-agenda-view>`:

```ts
html`
  <bc-calendar .localizer=${localizer} .events=${events}>
    <bc-default-toolbar></bc-default-toolbar>
    <bc-month-view></bc-month-view>
    <bc-time-grid-view></bc-time-grid-view>
    <my-agenda-view></my-agenda-view>
  </bc-calendar>
`
```

## Empty state

When there are no events in the agenda range, `<bc-agenda-view>` renders a `.bc-agenda-empty` element. The range length defaults to 30 days and is controlled by the `.length` property on `<bc-calendar>`.

Style the empty state with CSS:

```css
.bc-agenda-empty {
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  padding: 4rem 2rem;
  color: #888;
  font-size: 0.9em;
}
```

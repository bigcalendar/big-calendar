# Toolbar

The default toolbar provides **navigation** (Prev / Today / Next buttons) and a **view switcher** (one button per enabled view). It lives outside the `.bc-calendar` element so its height does not shrink the calendar's scrollable body.

## Hiding the toolbar

Pass `:toolbar="false"` to `<Calendar>` to remove it:

```vue
<Calendar :toolbar="false" />
```

Useful when your application provides its own navigation controls or when embedding a calendar widget without chrome.

## Customizing labels and messages

The toolbar reads from the `:messages` prop on `CalendarProvider`. Override any string:

```vue
<CalendarProvider
  :messages="{
    today: 'Jump to today',
    previous: '‹',
    next: '›',
    month: 'Month',
    week: 'Week',
    work_week: 'Work',
    day: 'Day',
    agenda: 'Agenda',
  }"
>
```

## Replacing the toolbar entirely

Provide your own component via `:components="{ toolbar: MyToolbar }"`. In Vue, the toolbar component receives its data as props:

```ts
import type { ToolbarProps } from '@big-calendar/vue'

const MyToolbar = {
  props: ['label', 'view', 'views', 'messages', 'onNavigate', 'onView'],
  template: `
    <header>
      <button type="button" @click="onNavigate('PREV')">Previous</button>
      <span>{{ label }}</span>
      <button type="button" @click="onNavigate('NEXT')">Next</button>
      <button type="button" @click="onNavigate('TODAY')">Today</button>
      <nav>
        <button
          v-for="v in views"
          :key="v"
          type="button"
          @click="onView(v)"
          :aria-pressed="v === view"
        >{{ messages[v] ?? v }}</button>
      </nav>
    </header>
  `,
}
```

```vue
<template>
  <CalendarProvider :components="{ toolbar: MyToolbar }">
    <Calendar />
  </CalendarProvider>
</template>
```

## Using the Toolbar component standalone

`<Toolbar>` is exported and can be placed anywhere inside a `CalendarProvider` — it does not have to be inside `<Calendar>`. This is how the `Calendar` component itself uses it:

```vue
<template>
  <CalendarProvider ...>
    <Toolbar />           <!-- navigation lives here, outside .bc-calendar -->
    <div class="bc-calendar">
      <MonthView />
      <TimeGridView />
      <AgendaView />
    </div>
  </CalendarProvider>
</template>
```

If you are building a custom layout with the headless composables (`useMonthView`, `useTimeGridView`, `useAgendaView`), this gives you full control over where navigation sits relative to the calendar body.

# Drag & Drop

Dragging an event to a new day or slot is really an **edit** of that event. Big Calendar splits this into clear jobs:

- **`@big-calendar/core`** does the **date math** ŌĆö given where you dropped the event, it works out the new start/end (keeping its duration).
- **`@big-calendar/dnd`** does the **pointer work** ŌĆö it turns the browser drag into "this event was dropped here" using [Pragmatic Drag and Drop](https://atlassian.design/components/pragmatic-drag-and-drop/about).
- **You** do the **saving** ŌĆö the calendar tells you the proposed change; your code writes it to your data.

This is the same idea as a controlled input: the calendar shows what you give it and tells you what the user did, but **you own the data**.

## Installation

```bash
pnpm add @big-calendar/dnd
```

## Turning it on

In Vue, use `shallowRef` (not `useRef`) and pass it to `useCalendarDnd`. Wrap your `<Calendar>` in a div that receives the ref:

```vue
<script setup>
import { ref, shallowRef } from 'vue'
import { defineComponent, h } from 'vue'
import {
  CalendarProvider,
  Calendar,
  useCalendarDnd,
} from '@big-calendar/vue'

// The DraggableCalendar component wires the DnD composable
const DraggableCalendar = defineComponent({
  name: 'DraggableCalendar',
  setup() {
    const calendarRef = shallowRef(null)
    useCalendarDnd(calendarRef)
    return () => h('div', { ref: calendarRef, style: 'display: contents' }, [h(Calendar)])
  },
})

const events = ref(myEvents)

const onEventDrop = ({ event, start, end, allDay }) => {
  events.value = events.value.map((e) =>
    e.id === event.id ? { ...e, start, end, allDay } : e,
  )
}
</script>

<template>
  <CalendarProvider :events="events" :onEventDrop="onEventDrop">
    <DraggableCalendar />
  </CalendarProvider>
</template>
```

`useCalendarDnd` must be called **inside** a component that is a descendant of `CalendarProvider`.

## What `onEventDrop` gives you

```ts
const onEventDrop = ({ event, start, end, allDay }) => {
  // event       ŌĆö your original event object, untouched
  // start / end ŌĆö the proposed new times, as ISO strings
  // allDay      ŌĆö whether it's now a whole-day event
}
```

Two things worth noting:

1. **It's a report, not a change.** The calendar never edits your events. You apply the new bounds.
2. **You get the original event**, so undoing is trivial.

## Resizing

In **time-grid** views, hovering a timed event reveals grab bars on its top and bottom edges. Drag an edge to resize ŌĆö the dragged edge snaps to the slot, the other stays put. A dashed live preview shows the proposed size while you drag.

In the **month** grid, hovering a segment reveals leading and trailing grab strips. Drag to grow or shrink by whole days.

Resize reports through `onEventResize`, which has the **exact same shape** as `onEventDrop`:

```vue
<script setup>
const apply = ({ event, start, end, allDay }) =>
  events.value.map((e) => (e.id === event.id ? { ...e, start, end, allDay } : e))
</script>

<template>
  <CalendarProvider :events="events" :onEventDrop="apply" :onEventResize="apply">
    <DraggableCalendar />
  </CalendarProvider>
</template>
```

## Async save with rollback

The standard real-world pattern: move the event immediately (optimistic update), then save; if the save fails, put it back.

```vue
<script setup>
const onEventDrop = async ({ event, start, end, allDay }) => {
  const previous = [...events.value]   // snapshot before the move

  events.value = events.value.map((e) =>
    e.id === event.id ? { ...e, start, end, allDay } : e,
  )

  try {
    await saveEvent(event.id, { start, end, allDay })
  } catch {
    events.value = previous            // rollback on failure
  }
}
</script>
```

The calendar never awaits your handler and never holds a copy of your data ŌĆö there is exactly one source of truth.

## Background events in drag and drop

When `backgroundEvents` are configured, every drop and resize callback includes a `backgroundEvents` field listing background events whose time range overlaps the new bounds. The field is **omitted entirely** when nothing overlaps.

```vue
<script setup>
import { getContainingBgEvents } from '@big-calendar/core'

const onEventDrop = (args) => {
  const { event, start, end, allDay, backgroundEvents } = args

  if (!backgroundEvents) {
    // ŌæĀ No overlap ŌĆö event landed entirely outside every background event
    alert('Please drop the event within an available block')
    return
  }

  const containers = getContainingBgEvents({ selection: args })

  if (containers.length > 0) {
    // ŌæĪ Fully contained ŌĆö event fits completely inside at least one background event
    events.value = events.value.map((e) =>
      e.id === event.id ? { ...e, start, end, allDay } : e,
    )
  } else {
    // Ōæó Partial overlap ŌĆö event crosses the edge of a background event
    alert('Please drop the event completely within an available block')
  }
}
</script>
```

`getContainingBgEvents` accepts the drop args object directly as `selection`. `onEventResize` and `onDropFromOutside` follow the exact same pattern.

## Locking events

Dragging and resizing are **opt-in** by default. Big Calendar reads `event.draggable` and `event.resizable` fields ŌĆö an absent or falsy value locks the event in place:

```ts
{ id: 1, title: 'Team sync', start: 'ŌĆ”', end: 'ŌĆ”', draggable: true, resizable: true }
{ id: 2, title: 'All-hands',  start: 'ŌĆ”', end: 'ŌĆ”' } // no fields ŌåÆ locked
```

Override the field names or use a function via `accessors.draggable` / `accessors.resizable`:

```vue
<CalendarProvider
  :accessors="{
    draggable: (e) => !e.locked,
    resizable: 'canResize',
  }"
/>
```

## Promoting a timed event to all-day

Drag any **timed** event from the time grid and drop it onto the **all-day row** above the time slots. The event becomes a whole-day event on the dropped day: `onEventDrop` fires with `allDay: true`.

**One-way only.** You can promote a timed event to all-day, but you cannot drag an all-day event down into the time grid.

## Dropping items from outside the calendar

Drag something from outside the calendar ŌĆö a palette of event templates, an unscheduled task list ŌĆö onto a time-grid slot or month day. This fires `onDropFromOutside` and **you** create the new event:

```vue
<CalendarProvider
  :onDropFromOutside="({ start, end, allDay }) => {
    events.value = [...events.value, { id: crypto.randomUUID(), title: 'New', start, end, allDay }]
  }"
>
```

The drag source carries a **payload** describing the event to create. Write the payload as JSON under `EXTERNAL_MIME` in the `dragstart` handler:

```vue
<script setup>
import { EXTERNAL_MIME } from '@big-calendar/dnd'

const onDragStart = (e) => {
  e.dataTransfer.setData(EXTERNAL_MIME, JSON.stringify({ durationMinutes: 60 }))
}
</script>

<template>
  <div draggable="true" @dragstart="onDragStart">1-hour block</div>
</template>
```

See the **Drag & Drop ŌåÆ Drop from Outside** story for a live demo.

## Dragging events out of the calendar

Every event automatically exposes its data on the native `dataTransfer` under the `EVENT_MIME` type, so any HTML5 dropzone outside the calendar can receive it:

```vue
<script setup>
import { EVENT_MIME } from '@big-calendar/dnd'

const onDrop = (e) => {
  const { id } = JSON.parse(e.dataTransfer.getData(EVENT_MIME))
  events.value = events.value.filter((ev) => String(ev.id) !== id)
}
</script>

<template>
  <div @dragover.prevent @drop="onDrop">Unschedule bin</div>
</template>
```

`onEventDragStart` fires when a drag begins (useful for highlighting valid drop zones).

# Custom Components

Every visual piece of the calendar is replaceable via the `:components` prop on `CalendarProvider`. In Vue, a "component" is any object with a `props` declaration and a `template` string (or a `setup` render function) — no special base class required.

More specific (per-view) slots always take precedence over shared ones. You can override as many or as few as you like; unset slots use the built-in defaults.

```vue
<script setup>
const MyMonthEvent = {
  props: ['event', 'title'],
  template: `<span :style="{ color: event.color }">{{ title }}</span>`,
}

const components = {
  toolbar: MyToolbar,
  month: { event: MyMonthEvent },
  time:  { event: MyTimeEvent },
}
</script>

<template>
  <CalendarProvider :components="components">
    <Calendar />
  </CalendarProvider>
</template>
```

---

## Vue vs React

In React, a slot component is a function. In Vue, it is a **component object** — the same kind of thing you pass to `defineComponent` or use as a plain object with `{ props, template }`. The pattern is:

```ts
// Simple: plain object with template string
const BoldWeekday = {
  props: ['long', 'day', 'short'],
  template: `<strong>{{ long }}</strong>`,
}

// When you need reactive setup logic:
import { defineComponent, h } from 'vue'
const CustomShowMore = defineComponent({
  props: { count: Number, events: Array, eventSlot: Object },
  setup(props) {
    return () => h('button', `+${props.count}`)
  },
})
```

The `eventSlot` prop in `MonthShowMoreProps` and `TimeShowMoreProps` is a Vue component — render it with `<component :is="eventSlot" :event="item.event" :title="item.title" />`.

## Month view slots (`components.month`)

### `weekday`

A column heading in the weekday header row (Sun, Mon, Tue, …).

```ts
const MyWeekday = {
  props: ['day', 'long', 'short'],
  template: `<abbr :title="long">{{ short }}</abbr>`,
}
```

### `dateCell`

The date header inside a day cell — the number, today/off-range state, and drill-down handler.

```ts

  props: ['label', 'isToday', 'isOffRange', 'onDrillDown', 'day'],

    <button type="button" @click="onDrillDown" :aria-label="'Drill down to ' + label">
      {{ label }}
    </button>
  `,
}
```

### `showMore`

The "+N more" overflow indicator shown when a week row exceeds `weekEventLimit`. The `eventSlot` prop is the currently configured month event component — use it to render each hidden event consistently.

```ts
import { defineComponent, h } from 'vue'

const MyShowMore = defineComponent({
  props: { count: Number, label: String, day: String, events: Array, eventSlot: Object },
  setup(props) {
    return () =>
      h('details', [
        h('summary', props.label),
        ...props.events.map((item) =>
          h(props.eventSlot, { key: item.key, event: item.event, title: item.title }),
        ),
      ])
  },
})
```

  <thead><tr><th>Prop</th><th>Type</th><th>Description</th></tr></thead>
  <tbody>
    <tr><td><code>count</code></td><td><code>number</code></td><td>How many events overflowed.</td></tr>
    <tr><td><code>label</code></td><td><code>string</code></td><td>Localized label ("+2 more").</td></tr>
    <tr><td><code>day</code></td><td><code>string</code></td><td>Day-start string of the overflow anchor.</td></tr>
    <tr><td><code>events</code></td><td><code>ReadonlyArray&lt;ShowMoreEvent&lt;TEvent&gt;&gt;</code></td><td>The hidden events.</td></tr>
    <tr><td><code>eventSlot</code></td><td><code>Component</code></td><td>The configured month event slot — use it to render each hidden event consistently.</td></tr>

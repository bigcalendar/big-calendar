# Dialog

A modal dialog built on the browser's native `<dialog>` element, rendered in the browser's top layer with a backdrop. Unlike `<Popover>`, a dialog is modal — it traps focus until closed and applies a semi-transparent backdrop behind it.

## Where it is used

Big Calendar exports `<Dialog>` as a building block for custom slot components that need a modal panel — for example, a "create event" or "edit event" form that opens when the user clicks a slot or event. The library does not open any dialogs internally; that is intentionally left to your application.

## API

```ts
import { Dialog } from '@big-calendar/vue'
```

In Vue, `Dialog` exposes an `open` prop and emits `close`. Content goes in the default slot:

```vue
<script setup>
import { ref } from 'vue'
import { Dialog } from '@big-calendar/vue'

const event = ref(null)
const close = () => { event.value = null }
</script>

<template>
  <Dialog :open="!!event" @close="close">
    <h2>Edit event</h2>
    <form method="dialog">
      <!-- your form fields -->
      <button>Save</button>
      <button type="button" @click="close">Cancel</button>
    </form>
  </Dialog>
</template>
```

## Styling

The base class is `bc-dialog`. The backdrop uses:

```css
.bc-dialog::backdrop {
  background: rgb(0 0 0 / 0.32);
}
```

Override either via your own CSS.

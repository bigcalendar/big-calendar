# `useFloatingAnchor`

Keeps a top-layer floating element positioned against an anchor element while it is open, repositioning automatically on scroll and window resize.

`<Popover>` and `<Tooltip>` both use this composable internally. It is exported so you can wire the same positioning behaviour into a **custom top-layer surface** — for example, a custom event details panel that uses the native Popover API or a modal `<dialog>`.

---

## When to use this

Use `useFloatingAnchor` when you:

- Have an anchor element (a button, an event box, a date cell).
- Have a floating element that uses the native `popover` attribute or `<dialog>` (so it renders in the browser's top layer).
- Want the floating element to stay aligned to the anchor as the page scrolls or the window resizes.

If you just need to compute a position once — without lifecycle management — use `positionFloating` directly (see **Utilities → Event Time**).

---

## Signature

```ts
import { useFloatingAnchor } from '@big-calendar/vue'

useFloatingAnchor(
  open:        Ref<boolean>,
  anchorRef:   Ref<HTMLElement | null>,
  floatingRef: Ref<HTMLElement | null>,
  placement?:  FloatingPlacement,
  sameWidth?:  boolean,
): void
```

The composable returns nothing. It manages positioning as a side effect.

In Vue, pass reactive `Ref` values (from `ref()` or `shallowRef()`) rather than plain booleans or DOM nodes — the composable watches `open` to add and remove scroll/resize listeners.

## Usage example

```vue
<script setup>
import { ref, shallowRef } from 'vue'
import { useFloatingAnchor } from '@big-calendar/vue'

const props = defineProps(['event', 'title'])

const open = ref(false)
const anchorRef = shallowRef(null)
const panelRef = shallowRef(null)

useFloatingAnchor(open, anchorRef, panelRef, 'bottom-start')
</script>

<template>
  <button :ref="anchorRef" @click="open = !open">
    {{ title }}
  </button>
  <div
    v-if="open"
    :ref="panelRef"
    popover="auto"
    style="position: fixed; margin: 0"
  >
    <h3>{{ title }}</h3>
    <p>{{ event.start }} – {{ event.end }}</p>
    <button @click="open = false">Close</button>
  </div>
</template>
```

### What happens on each positioning run

1. `positionFloating` (from `@big-calendar/core/utils`) is called with the anchor and floating elements and the requested placement.
2. The result `{ x, y }` is applied as `position: fixed; inset-inline-start: Xpx; inset-block-start: Ypx`.
3. The browser's UA stylesheet for `[popover]` sets `inset: 0` on all four sides by default. The composable resets `inset-block-end` and `inset-inline-end` to `auto` on every run so they do not stretch the panel to the viewport edge.
4. `window` `scroll` and `resize` listeners repeat the run while `open` is `true`, then remove themselves when `open` becomes `false` or the component unmounts.

---

## Relationship to `<Popover>` and `<Tooltip>`

`<Popover>` and `<Tooltip>` both call `useFloatingAnchor` and handle open/close state, trigger wiring, and ARIA attributes for you. Use them directly unless you need a surface that does not fit either of those patterns.

# useTimeGridView

A composed composable that drives a custom time-grid view (Day, Week, Work-Week). Call it inside a component that lives inside a `CalendarProvider`. See **Advanced → Headless API** for the general pattern.

```ts
import { useTimeGridView } from '@big-calendar/vue'
```

## Return value

## Layout variants

The time grid has three layout paths, identified by checking the `grid` object:

<table>
  <thead><tr><th>Condition</th><th>Layout</th></tr></thead>
  <tbody>
    <tr><td><code>grid.resources === null &amp;&amp; grid.dayGroups === null</code></td><td><strong>Plain</strong> — one column per day.</td></tr>
    <tr><td><code>grid.resources !== null</code></td><td><strong>Resource-major</strong> — resources as the outer grouping.</td></tr>
    <tr><td><code>grid.dayGroups !== null</code></td><td><strong>Day-major</strong> — days as the outer grouping (<code>resourceLayout="day"</code>).</td></tr>
  </tbody>
</table>

## Composed vs. granular composables

`useTimeGridView` wraps three lower-level composables:

1. **`useTimeGrid()`** — builds the grid model from the store.
2. **`useTimeGridHeader(grid)`** — header state, all-day-row interaction, roving focus, segment geometry.
3. **`useTimeGridBody(grid)`** — body state, time-slot interaction, event positioning, background events, now indicator.

Call those directly if you only need to replace the header or body, not the whole layout.

## Usage example (plain layout)

```vue
<script setup>
import { EventButton, useTimeGridView } from '@big-calendar/vue'

const { grid, announcement, root, getRootStyle, header, body } =
  useTimeGridView()

const {
  components: { DayHeading, AllDayEvent },
  timeHead, timeHeader, timeHeaderGutter,
  allDayRow, allDayLabel, allDaySlots, allDaySegments,
  getHeadingProps, getAllDaySlotProps, getAllDaySegmentProps,
} = header

const {
  components: { TimeLabel, EventSlot },
  body: bodyRoot, gutter, timeSlotsContainer,
  getColumnProps, getSlotProps, getEventProps,
  bodyNowIndicatorProps,
} = body
</script>

<template>
  <template v-if="grid !== null">
    <div :style="getRootStyle(grid.headings.length)" v-bind="root">
      <div role="status" aria-live="polite" class="bc-sr-only">
        {{ announcement }}
      </div>
      <div v-bind="timeHead">
        <div v-bind="timeHeader">
          <div v-bind="timeHeaderGutter" />
          <component
            v-for="h in grid.headings"
            :key="h.day"
            :is="DayHeading"
            v-bind="getHeadingProps(h)"
          />
        </div>
        <div v-bind="allDayRow">
          <div v-bind="allDayLabel">All day</div>
          <div v-bind="allDaySlots">
            <div
              v-for="(col, ci) in grid.columns"
              :key="col.key"
              v-bind="getAllDaySlotProps(col, ci)"
            />
          </div>
          <div v-bind="allDaySegments">
            <EventButton
              v-for="seg in grid.allDay.segments"
              :key="seg.key"
              v-bind="getAllDaySegmentProps(seg)"
            >
              <component :is="AllDayEvent" :event="seg.event" :title="seg.title" />
            </EventButton>
          </div>
        </div>
      </div>
      <div v-bind="bodyRoot">
        <div v-bind="gutter">
          <component
            v-for="l in grid.gutter"
            :key="l.key"
            :is="TimeLabel"
            :time="l.time"
            :label="l.label"
          />
        </div>
        <div
          v-for="(column, ci) in grid.columns"
          :key="column.key"
          v-bind="getColumnProps(column)"
        >
          <div v-bind="timeSlotsContainer">
            <div
              v-for="(_, si) in grid.slotCount"
              :key="si"
              v-bind="getSlotProps(column, ci, si)"
            />
          </div>
          <EventButton
            v-for="event in column.events"
            :key="event.key"
            v-bind="getEventProps(event)"
          >
            <component
              :is="EventSlot"
              :event="event.event"
              :title="event.title"
              :time="event.time"
            />
          </EventButton>
        </div>
        <div v-if="bodyNowIndicatorProps" v-bind="bodyNowIndicatorProps" />
      </div>
    </div>
  </template>
</template>
```

## When to use this vs. `<TimeGridView>`

Use `<TimeGridView>` when the default rendering is sufficient. Use `useTimeGridView` when you need to change the HTML structure — render the all-day row differently, replace the column layout, or add custom wrappers.

For changing individual cells or labels without restructuring, use `components.time` slot overrides on `CalendarProvider` — no composable required.
